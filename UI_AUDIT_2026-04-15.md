Walk-through of every page after the morning sprint (#225 button-tabs,
#226 SessionCard polish, #227 ProjectCard polish, plus the overnight

Severity:

- **Major** — affects usability, accessibility, or design coherence in a

## Cross-cutting findings

### #M-1 (Major) — ProjectsPage filter pills + Rescan button drift from design system

`ProjectsPage.tsx:81-114`. The four filter pills + Rescan button still
use inline `background`/`border`/`color` style props (active-pill uses a
flat `var(--color-accent)` background instead of the new `.cta-btn-primary`
gradient). Inconsistent with the `.nav-btn` / `.cta-btn-primary` /
`.session-tab` motion language now used everywhere else.

**Fix**: introduce a `.filter-chip` + `.filter-chip--active` class set in
`index.css` and migrate the filter pills. Rescan button → `.nav-btn--muted`
with a left RefreshCw icon. Filed as **#234**.

### #M-2 (Major) — ProjectDetailPage layout has not received the polish pass

`ProjectDetailPage.tsx`. Header still uses raw padding/sizes, the back
link is plain text, the handoff toggle is a `<button>` with inline
ChevronDown swap and no consistent treatment with the `.session-tab`
expand/collapse pattern. The PhaseTimeline / ModuleMap / StateViewer
children deserve a once-over too.

**Fix scope**: visual pass mirroring the SessionCard / ProjectCard
treatments — modern stroked surfaces, consistent typography scale,
accent glow on interactive sections. Filed as **#235**.

### #m-1 (Minor, FIXED) — TerminalPage loading state uses raw text

`TerminalPage.tsx:38-42` rendered "Loading sessions..." as plain text
instead of using `LoadingSkeleton`. Inconsistent with every other page.
**Fixed in this commit** — now renders `LoadingSkeleton variant="card"`.

### #m-2 (Minor, FIXED) — ProjectDetailPage loading state uses raw spinner

`ProjectDetailPage.tsx:52-58` rendered a centered `Loader2` spinner.
**Fixed in this commit** — now renders `LoadingSkeleton variant="card"`
matching the SessionsPage / ProjectsPage / AnalyticsPage convention.

## Accessibility findings

### Aria-label coverage

Searched `aria-label`/`aria-current`/`role=` across 16 occurrences in 10
files. Coverage is light but present at the high-traffic interactive
surfaces (sidebar nav, TopCommandBar tabs, SplitChatLayout panes,
SessionCard quick-split). 

**Gaps worth flagging** (Nit-level — can be addressed in a future a11y
sprint):

- StatusBadge has no aria-label or role; relies on visible text. Fine
  for sighted users; a screen reader would read e.g. just "working" out
  of context. A wrapping span with `aria-label="Session status: working"`
  would be friendlier.
- The pencil rename button on SessionCard has `title="Rename session"`
  but no `aria-label`. Lucide icon-only button → screen reader hears
  nothing useful.
- ProjectCard linked-sessions cluster has a `title` but no
  `role="status"` / `aria-label`.

Not filing as Major — these are common in modern dashboards. A focused
a11y audit (separate task) would handle them as a batch.

### Keyboard navigation

- `<button>` elements receive default focus rings.
- ESC / Cmd+. global interrupt works.
- Modal traps not verified (CreateSessionModal). Worth a focused test;
  filed as nit.
