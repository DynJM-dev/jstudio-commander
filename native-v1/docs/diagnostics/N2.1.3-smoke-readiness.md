### §1.4 — OSC 133 A marker firing (transitive proof)

`BootstrapLauncher.onOsc133` (apps/sidecar/src/pty/bootstrap.ts:119) is
gated on `event.marker !== 'A'` and writes `claude\n` only when an A marker
fires. Since Claude Code's TUI is demonstrably running in all three
spawned sessions, A fired in each → OSC 133 hook is correctly installed +
emitting.

(Sidecar emits `command:started` / `command:ended` / `prompt:started`
events on the `session:<id>` channel per orchestrator.ts. The smoke probe
connected late to that channel — events during spawn went to buffered
subscriptions. Direct observation of A markers would require a timing-
adjusted WS subscribe-before-spawn probe. Given the transitive proof via
Claude boot, filed for Task 4 completeness not as a correctness concern.)
