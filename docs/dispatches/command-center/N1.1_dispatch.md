## §1 — What failed

N1 smoke step 2 (Finder launch) failed: double-click is a silent no-op. No process spawn, no crash dialog, no Gatekeeper prompt. PM diagnosis complete in `PHASE_N1_REPORT.md` §3.3 — no instrumentation rotation required; evidence is load-bearing and the fix is direct.

**Root cause:** `Contents/_CodeSignature/CodeResources` missing from the bundle. `spctl -a -vvv` reports *"code has no resources but signature indicates they must be present."* `commander-shell` has a linker-injected ad-hoc Mach-O signature (standard arm64) but no bundle-level `codesign --force --deep --sign -` pass was run. macOS Sequoia + Apple Silicon's strict verification rejects the inconsistent state silently. This is a gap in the Bun-workspace `bun run build:app` pipeline — v1's `pnpm tauri build` invoked bundle codesign implicitly; CODER's N1 scaffold does not.

**Secondary:** native-v1 archive + N1 build share bundle ID `studio.jstudio.commander`. Launch Services keys by bundle ID; collision is real but not primary cause.
