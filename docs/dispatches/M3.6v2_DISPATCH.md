macOS has a case-insensitive filesystem. When CTO wrote `~/Desktop/Projects/JSTUDIO/` in every document this session, macOS silently resolved that path to `~/Desktop/Projects/JStudio/` — the agency website project folder. Every "ops file" shipped this session (OS v1.2, templates, standards files, migration state, playbook) landed inside the agency website directory instead of a dedicated ops folder.

Three distinct JStudio-prefixed folders currently exist and two of them differ only by case:

```
~/Desktop/Projects/JStudio/          ← agency website (Firebase project) + misplaced ops files
~/Desktop/Projects/jstudio-master/   ← JStudio Core ERP template (active, used as seed for new ERP projects)
~/Desktop/Projects/jstudio-commander/ ← Command Center (unaffected)
```

The case collision between `JStudio` and `JSTUDIO` is the root cause of this bug. It will happen again if not fixed structurally.

for file in "${FILES[@]}"; do
  if [ ! -f "$file" ]; then
    echo "MISSING: $file"
    MISSING_COUNT=$((MISSING_COUNT+1))
    continue

     grep -q "Desktop/Projects/JStudio/" "$file" 2>/dev/null; then

    CHANGED_COUNT=$((CHANGED_COUNT+1))
  else
    echo "SKIPPED (no matches): $file"
    SKIPPED_COUNT=$((SKIPPED_COUNT+1))
  fi
done

echo ""
echo "Summary: $CHANGED_COUNT updated, $SKIPPED_COUNT skipped, $MISSING_COUNT missing"

## 10. Step 8 — Create the case-collision guard script

Prevent this class of bug from recurring.

```bash
mkdir -p ~/Desktop/Projects/jstudio-meta/scripts/

cat > ~/Desktop/Projects/jstudio-meta/scripts/check-case-collisions.sh <<'EOF'
#!/bin/bash
# Check for directory names that differ only by case.
# On case-insensitive filesystems (macOS default), such collisions cause
# silent path resolution bugs.
#
# Usage: ./check-case-collisions.sh [directory]
# Default directory: ~/Desktop/Projects/

DIR="${1:-$HOME/Desktop/Projects}"

if [ ! -d "$DIR" ]; then
  echo "ERROR: Directory $DIR does not exist"
  exit 1
fi

# Find directories that collide when lowercased
COLLISIONS=$(ls -1 "$DIR" 2>/dev/null | awk '{print tolower($0), $0}' | sort | \
  awk 'prev_lower == $1 { print prev_orig "\n" $2 } { prev_lower = $1; prev_orig = $2 }')

if [ -z "$COLLISIONS" ]; then
  echo "✓ No case-collision risks found in $DIR"
  exit 0
else
  echo "⚠ CASE COLLISION RISK in $DIR:"
  echo "$COLLISIONS" | sort -u
  echo ""
  echo "On case-insensitive filesystems (default macOS), these names may resolve"
  echo "to the same directory depending on how paths are typed. Rename to"
  echo "disambiguate before this causes a bug."
  exit 1
fi
EOF

chmod +x ~/Desktop/Projects/jstudio-meta/scripts/check-case-collisions.sh

# Run it to verify it works
~/Desktop/Projects/jstudio-meta/scripts/check-case-collisions.sh

# Expected output: "✓ No case-collision risks found in /Users/.../Desktop/Projects"
```

Commit the script:

```bash
cd ~/Desktop/Projects/jstudio-meta/
git add scripts/
git commit -m "chore: add case-collision guard script (prevents the M3.6 bug class)"
```

Optional: Jose can add this to a cron job or shell login script if he wants automatic checking. Skip unless Jose requests it.
